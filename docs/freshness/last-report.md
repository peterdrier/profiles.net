The prune agents surfaced 5 borderline items they would not assert as migrations (each either borders on restating obvious behavior or fits none of the three allowed destinations). Proposed text below; apply next pass if you judge them durable:

1. **Facilitated-messaging button visibility** → `docs/sections/Profiles.md` (Invariants):
   > The "Send a message" (facilitated messaging) button on a profile card appears only when the viewer is not the profile owner and cannot see any email for that human (no visible `UserEmail` and no Email-type `ContactField` at the viewer's visibility level) — when an email is already visible, the viewer contacts them directly.

2. **Campaign unsubscribe token mechanism** → `docs/sections/Campaigns.md` (or Users section):
   > Unsubscribe tokens are minted via `IDataProtectionProvider` time-limited protector (`"CampaignUnsubscribe"`) with a 90-day expiry; expired tokens render a friendly "link expired" page, invalid tokens 404.

3. **"Coordinator" untranslated** → `docs/sections/Teams.md` (Concepts):
   > The term "Coordinator" is kept untranslated (English) across all locales, mirroring the untranslated "humans" convention.

4. **Team-member departure single-`SaveChanges` EF ordering gotcha** → `docs/architecture/conventions.md` (Transaction Boundary) or `Teams.md`:
   > Member departure (`LeaveTeamAsync`/`RemoveMemberAsync`) deletes the departing member's `TeamRoleAssignment` rows and sets `TeamMember.LeftAt` in a single `SaveChangesAsync` — EF issues DELETEs before UPDATEs in one save, satisfying the Restrict FK on `TeamRoleAssignment.TeamMember`. Do not split into separate saves.

5. **Service-layer validation over DB CHECK constraints** → `docs/architecture/conventions.md` (candidate convention): the role-slots spec stated `SlotIndex < SlotCount` is enforced in the service layer with no DB CHECK constraint, "consistent with the project's preference for service-layer validation over database constraints."
